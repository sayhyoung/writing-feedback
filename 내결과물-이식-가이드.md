# '내 결과물' 로컬 보관함 이식 가이드

WriteBack 프로젝트에 구현한 '내 결과물' 로컬 보관함을 다른 사이트(예: 문법 문항 생성 사이트)에
그대로 이식하기 위한 지시서입니다. 다른 개발 세션이나 개발자에게 그대로 전달할 수 있습니다.

---

## 1. 배경·목표

WriteBack 프로젝트에 구현한 '내 결과물' 로컬 보관함을 이 사이트에도 똑같이 붙이고 싶습니다.
서버나 로그인 없이 **브라우저 IndexedDB에 기기별로** 결과를 저장하고,
**우측 상단 메뉴 아이콘**으로 여는 패널에서 과거 결과를 다시 보고·삭제할 수 있게 합니다.

## 2. 저장 방식 (핵심 결정사항 — 꼭 명시)

- **저장소**: `IndexedDB` (DB 1개, objectStore 1개, `keyPath: 'id'`, `autoIncrement: true`).
  localStorage 아님 — 용량·구조화 때문.
- **서버 저장/로그인/DB 비용 없음.** 순수 클라이언트, 기기별 로컬.
  → 다른 기기에서는 안 보임 (이 점을 사용자에게 안내 문구로 표기).
- **성공한 결과만 저장.** 실패/빈 결과 메시지는 저장하지 않도록 실패 문자열 목록으로 가드.
- 진입점은 **탭이 아니라 우측 상단 아이콘 버튼** (자주 쓰는 기능이 아니므로).

## 3. 저장할 레코드 스키마 (도메인에 맞게 필드만 교체)

WriteBack에서는 이런 형태로 저장했습니다:

```js
{
  id,                    // 자동 증가 (자동)
  createdAt: Date.now(), // 정렬·표시용
  name, school,          // 식별용 메타 (→ 그 사이트에 맞는 필드로 교체)
  originalText,          // 입력 원문   (→ 예: 생성 조건/난이도/지문)
  correctedHtml,         // 결과 본문 HTML (→ 예: 생성된 문항 HTML)
  correctedText,         // 결과 본문 텍스트
  // ↓ 부가 자산이 있으면 (WriteBack은 음원). 없으면 이 3개 생략
  audioBase64: null, timepoints: [], sentences: [],
}
```

> **문항 생성 사이트 기준 예시 필드**: `createdAt`, `name`, `conditions`(생성 조건),
> `questionsHtml`(생성된 문항), `answerKey`(정답/해설) 정도로 매핑하면 됩니다.

## 4. 붙일 코드 4덩어리 (WriteBack에 이미 구현됨 — 그대로 참고)

1. **IndexedDB 헬퍼**: `openHistoryDB / historyAdd / historyUpdate / historyGetAll /
   historyGet / historyDelete` (`public/script.js` 66~137행).
   도메인 무관하게 **그대로 복사** 가능.
2. **저장 트리거**: 결과 생성 성공 직후 `historyAdd({...})` 호출하고 반환된 id를
   `currentResultId`에 보관. 실패 문자열 가드 포함 (2140~2165행 패턴).
3. **패널 UI**: 우측 상단 아이콘 버튼(`history-btn`) → 클릭 시 모달 오버레이.
   목록(최신순, 메타·날짜·미리보기·열기/삭제) → 상세보기(본문 재표시 + 다시 다운로드/인쇄).
   (`openHistoryPanel / renderHistoryList / openHistoryDetail`, 988~1090행)
4. **CSS**: `.topbar`, `.history-btn`, `.history-dialog`, `.history-list`,
   `.history-item`, `.history-detail` 등 (style.css). 색상만 그 사이트 톤에 맞게.

## 5. 주의사항 (그대로 전달)

- 결과가 나중에 **편집**되면 저장 레코드도 `historyUpdate`로 동기화
  (부가 자산 캐시는 무효화).
- 부가 자산(WriteBack의 음원 같은 것)이 있다면 **생성 시점에 실제로 만든 경우에만**
  레코드에 추가 저장 — 미생성 시 "저장된 ○○ 없음" 안내.
- HTML을 다시 그릴 때 XSS 방지용 이스케이프(`escHtml`) 사용.

## 참고: 재사용 가능한 IndexedDB 헬퍼 (그대로 복사)

```js
// ===== IndexedDB: 로컬 "내 결과물" 보관함 =====
const HISTORY_DB = 'writeback-history';   // → 사이트별 고유 이름으로 변경
const HISTORY_STORE = 'results';
let currentResultId = null; // 현재 화면에 표시 중인 결과의 저장 id

function openHistoryDB() {
    return new Promise((resolve, reject) => {
        const req = indexedDB.open(HISTORY_DB, 1);
        req.onupgradeneeded = () => {
            const db = req.result;
            if (!db.objectStoreNames.contains(HISTORY_STORE)) {
                db.createObjectStore(HISTORY_STORE, { keyPath: 'id', autoIncrement: true });
            }
        };
        req.onsuccess = () => resolve(req.result);
        req.onerror = () => reject(req.error);
    });
}
async function historyAdd(record) {
    const db = await openHistoryDB();
    return new Promise((resolve, reject) => {
        const tx = db.transaction(HISTORY_STORE, 'readwrite');
        const req = tx.objectStore(HISTORY_STORE).add(record);
        req.onsuccess = () => resolve(req.result);
        req.onerror = () => reject(req.error);
    });
}
async function historyUpdate(id, patch) {
    if (id == null) return;
    const db = await openHistoryDB();
    return new Promise((resolve, reject) => {
        const tx = db.transaction(HISTORY_STORE, 'readwrite');
        const store = tx.objectStore(HISTORY_STORE);
        const getReq = store.get(id);
        getReq.onsuccess = () => {
            const rec = getReq.result;
            if (!rec) { resolve(); return; }
            Object.assign(rec, patch);
            const putReq = store.put(rec);
            putReq.onsuccess = () => resolve();
            putReq.onerror = () => reject(putReq.error);
        };
        getReq.onerror = () => reject(getReq.error);
    });
}
async function historyGetAll() {
    const db = await openHistoryDB();
    return new Promise((resolve, reject) => {
        const tx = db.transaction(HISTORY_STORE, 'readonly');
        const req = tx.objectStore(HISTORY_STORE).getAll();
        req.onsuccess = () => resolve(req.result || []);
        req.onerror = () => reject(req.error);
    });
}
async function historyGet(id) {
    const db = await openHistoryDB();
    return new Promise((resolve, reject) => {
        const tx = db.transaction(HISTORY_STORE, 'readonly');
        const req = tx.objectStore(HISTORY_STORE).get(id);
        req.onsuccess = () => resolve(req.result || null);
        req.onerror = () => reject(req.error);
    });
}
async function historyDelete(id) {
    const db = await openHistoryDB();
    return new Promise((resolve, reject) => {
        const tx = db.transaction(HISTORY_STORE, 'readwrite');
        const req = tx.objectStore(HISTORY_STORE).delete(id);
        req.onsuccess = () => resolve();
        req.onerror = () => reject(req.error);
    });
}
```

## 참고: 성공 결과만 저장하는 가드 패턴

```js
currentResultId = null;
const modelAnswer = (data.modelAnswer || '').trim();
const FAILURE_ANSWERS = [
    '수정글 생성에 실패했습니다. 피드백 내용을 참고해주세요.',
    '피드백을 제공할 영어 문장이 없습니다. 영작문을 입력해주세요.',
    // → 그 사이트의 실패/빈 결과 메시지로 교체
];
const isValidResult = modelAnswer && !FAILURE_ANSWERS.includes(modelAnswer);
if (isValidResult) {
    try {
        currentResultId = await historyAdd({
            createdAt: Date.now(),
            // ... 사이트 도메인에 맞는 필드
        });
    } catch (_) { currentResultId = null; }
}
```
